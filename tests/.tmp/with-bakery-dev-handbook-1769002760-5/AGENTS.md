# 스크립트

또한 사용자의 지시 없이는 절대적으로 기존 **코딩 스타일**을 바꾸지 않습니다.
예시:
`if-then-else` 를 삼항 연산자로 바꾸는 일을 하지 않습니다.
`for i, v in t do` 에 함부로 `pairs` 를 넣지 않습니다.

## 주석

주석은 한글로 적습니다
매번 블록주석으로 모듈 위에 무슨 모듈인지 서술해야합니다.

## 출력

출력은 영어로 적습니다.

`assert` 나 `error` 의 과다 사용을 자제합니다.
타입 검증을 위한 코드, if 문 등을 사용자의 지시가 없는한 작성하지 않습니다.
잘못된 인수를 주는 것을 대비하는 것보다, 타입을 명확히 설정하는 것이 낫습니다.

유저의 특정한 요구가 없는 이상 추가적인 `assert` 나 `error` 를 첨부하지 않습니다.

## 에러가 날 수 있는 잠재적 상황에서
`error` 나 `if`, `typeof` 로 분기를 나누는 것 보다
그냥 작성자가 타입을 제대로 줬다고 믿고, 작성합니다.
과도한 if 문과 분기는 코드를 복잡하게 만듭니다.

## 재사용
이미 존재하는 공용 유틸리티가 동일한 목적을 제공한다면 지역 수준의 헬퍼 함수를 새로 작성하지 않고 우선 재사용합니다.

# 가이드 무조건 준수
이 레포지토리에서 작업을 할 때는 [guides](libs/bakery-dev-handbook/docs/guides/) 폴더 내 내용을 **무조건** 준수해야합니다.

그러나 가이드에 적힌 내용을 **과도 해석** 해서 적히지 않은 관습을 절대로 추가하지 않습니다.
문서에 정의되지 않은 키워드(예: moonwave `@module` 태그)나 구문은 추가하지 않습니다.


# 스크립트 및 설계도 작성
코딩 시 다음 사항을 **무조건** 지켜야합니다.

모든 AGENT 는 작업 전 다음 문서를 전부 꼼꼼히 읽어야합니다.

문서 속 프로젝트 지침, [frameworks](libs/bakery-dev-handbook/docs/guides/frameworks) 를 **절대적으로** 준수할 것.

작업시작 전 해당 문서를 무조건적으로 꼼꼼히 읽어봐야하며 함부로 자신만의 스타일로 만들지 않습니다.
문서에 정의된 스타일로만 코드를 작성해야합니다.



특정 용어가 나왔을 경우 해당 문서를 찾아 공부합니다.
